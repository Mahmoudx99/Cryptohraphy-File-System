Don't attempt to move the .ipynb script from the orignal directory.
If you do proceed at your own risk.
The code get initially generate directory tree that works on, however later on it will get all files in the subdirectories
of the current path. for example if you put it in the desktop it will encrypt all the files under its path.

Be cautious, run the code within it's current directory [cryptographic file system], it will generate directory tree named [test_directory].

In the GUI, run the setup.py file and put the path of your python. 
the file will create build folder which will contain the excutable file.
